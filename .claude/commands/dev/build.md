프로덕션 빌드를 실행하고, 발생하는 모든 에러를 수정합니다.

# Process

## Step 1: 빌드 실행

```bash
npm run build
```

빌드를 실행하고 출력을 확인합니다.

## Step 2: 에러 분석

빌드 실패 시:
- TypeScript 타입 에러 확인
- ESLint 에러 확인
- Import 경로 에러 확인
- 빌드 타임 에러 확인

## Step 3: 에러 수정

발견된 에러를 하나씩 수정합니다:

1. **TypeScript 에러**
   - 타입 정의 추가/수정
   - 누락된 props 추가
   - 잘못된 타입 캐스팅 수정

2. **Import 에러**
   - 경로 수정
   - 누락된 export 추가
   - 순환 의존성 해결

3. **ESLint 에러**
   - 사용하지 않는 변수 제거
   - React hooks 규칙 준수
   - 접근성 규칙 준수

## Step 4: 재빌드 및 검증

에러 수정 후 다시 빌드를 실행하여 성공 여부를 확인합니다.

모든 에러가 해결될 때까지 Step 2-4를 반복합니다.

# Guidelines

- 에러 수정 시 기능이 깨지지 않도록 주의
- `any` 타입 사용 최소화
- 사용하지 않는 코드는 완전히 제거 (주석 처리 X)
- 빌드 경고도 가능한 해결

# Quality Checks

빌드 완료 전 확인:
- [ ] `npm run build` 성공
- [ ] TypeScript 에러 없음
- [ ] ESLint 에러 없음
- [ ] 콘솔 경고 최소화

# Output Format

```markdown

- [경고 내용]

- 번들 크기: XKB
```
